## 1. Security fix — `server.js` (this session)

- **Rate limiting now keys on the real client IP.** Previously used
  `req.socket.remoteAddress`, which behind Railway's proxy is the proxy's
  own IP for every visitor — meaning the 5-requests/hour limit on
  `/api/contact` was effectively shared globally instead of per-visitor.
  Added `clientIp()`, which reads the **last** entry of `X-Forwarded-For`
  (the one Railway's own proxy appends, not spoofable by the client) and
  falls back to `remoteAddress` when there's no proxy.
- **Email subject stripped of CRLF.** The contact-form subject line
  (`Nova mensagem de ${name}`) now has `\r`/`\n` replaced with spaces
  before being sent. Not currently exploitable — Resend is called via its
  JSON API, not hand-built SMTP headers — but this is defense-in-depth in
  case the sending path ever changes.
- Verified with a real Node runtime (v22.13.0, downloaded standalone into
  the session scratchpad since `node` isn't on this machine's `PATH`):
  syntax check passed, and both fixes were exercised live against a
  running instance of the server (rate-limit bucketing by trailing XFF
  entry, CRLF actually stripped from the subject).

## 2. Contact form robustness (server + client)

- Server (`server.js`): name must be ≥2 chars, email validated with a real
  regex (`algo@dominio.tld`) instead of just checking for `@`, notes must
  be ≥10 chars.
- Client (`booking.js`): mirrors the same rules for inline field
  validation (on blur, then live once a field has an error), with new
  PT/EN error strings and `.field-error` styling. Comment in the code is
  explicit that the server remains the source of truth — client validation
  is only for immediate feedback.
- `contactos.html`: inputs gained matching `minlength`/`maxlength`
  attributes.

## 3. Performance

- **Fonts self-hosted.** All Google Fonts `<link>`/`@font-face` CDN calls
  removed from every page; `styles.css` now declares `@font-face` rules
  pointing at `/fonts/*.woff2` (22 files, latin + latin-ext subsets for
  Archivo/Public Sans/IBM Plex Mono). Every page's `<head>` swapped the
  `preconnect`+stylesheet pair for two `<link rel="preload" as="font">`
  hints. `styles.css?v=16` bumped to `v17` everywhere consistently.
- **Server-side compression** (`server.js`): brotli or gzip for
  HTML/CSS/JS/JSON/SVG/XML/TXT responses, negotiated via `Accept-Encoding`.
- **Cache-Control tuned**: versioned assets (`?v=`) and `.woff2` get a
  1-year immutable cache; HTML/CSS/JS without a version stay `no-cache`;
  everything else moved from 1-day to 7-day cache.
- **Speculation Rules prefetch** (`<script type="speculationrules">`)
  added to every page, prefetching `/contactos` and `/servicos`.
- `prefers-reduced-motion: reduce` now disables the marquee animation and
  smooth scroll.

## 6. Privacy / Analytics — new `consent.js`

New file, **currently untracked**, referenced by `<script src="/consent.js"
defer>` on every page. Implements a GDPR-style hard-block cookie banner:
GA4's `gtag.js` is not requested at all until the visitor explicitly
accepts (no script, no cookie, no request to Google by default). Decision
persists in `localStorage`, reversible via a "Cookies" link injected into
the footer, re-renders in the visitor's current language.

**GA4 is currently a placeholder** — `GA4_ID = "G-XXXXXXXXXX"` in
`consent.js`. The code deliberately no-ops (logs to console, doesn't load
`gtag.js`) while the ID is unset, so nothing breaks, but analytics won't
actually collect anything until a real Measurement ID is filled in.

The CSP in `server.js` was updated to allow `googletagmanager.com` /
`google-analytics.com` (needed for `gtag.js` to load *after* consent) and
to drop the now-unused `fonts.googleapis.com` / `fonts.gstatic.com`
entries (fonts are self-hosted now, confirmed no remaining references
anywhere in `public/`).
